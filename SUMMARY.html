End-of-Turn Detection

This submission implements a causal end-of-turn detector for spoken conversation. At each candidate pause, the system predicts the probability that the speaker has completed their turn rather than temporarily hesitating.

Approach

The model uses only information available before the candidate pause. No future audio is accessed. The feature representation captures local prosodic behaviour and causal turn context.

Features include energy level and trajectory, pitch level and trajectory, pitch variability, voiced fraction, final voiced behaviour, elapsed turn time, pause position, and available speech context.

Model

The final classifier is a standardized Logistic Regression model with balanced class weights. Experiments also evaluated Histogram Gradient Boosting, but Logistic Regression achieved better out-of-fold performance on the primary Hindi latency metric.

The final architecture contains three saved models:

  • Hindi specialist
  • English specialist
  • Global fallback

The inference pipeline selects a specialist when the language can be reliably inferred from the dataset structure or identifiers and otherwise uses the global fallback model.

Validation Methodology

Model selection used five-fold grouped out-of-fold validation. Splits were grouped by turn ID so that pauses belonging to the same turn could not leak between training and validation.

Experiments

The initial baseline produced approximately 1600 ms response delay. The provided starter prosodic model improved the English result to 1190 ms.

Subsequent experiments introduced richer energy, pitch, voicing and turn-context features. Honest grouped out-of-fold evaluation was then introduced to avoid optimistic estimates caused by fitting and evaluating on the same turns.

Histogram Gradient Boosting, probability ensembling, and a pause-position probability heuristic were evaluated. None improved the primary Hindi response-delay metric over the final Logistic Regression approach.

Causality

All acoustic features are computed from audio at or before pause_start. The system does not inspect future speech when making an end-of-turn decision.

Inference

The final predict.py loads pre-trained models from disk and generates an EOT probability for every candidate pause. It does not retrain or refit on unseen evaluation data.

Key Finding

Simple causal prosodic features provide meaningful end-of-turn information. In this dataset, a regularized linear classifier generalized better than the tested nonlinear boosting model. Performance also differed substantially between Hindi and English, motivating language-specialized models with a global fallback.
